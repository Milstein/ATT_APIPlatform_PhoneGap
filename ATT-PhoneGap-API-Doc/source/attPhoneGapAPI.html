/**

 *
 * @param {Object} options Specifies an object containing the following parameters.
 * @param {String} options.clientId Specifies the API key that is assigned to the application when the developer creates an application account at <a href="https://developer.api.att.com">https://developer.api.att.com</a>.

 * <p>The defined values for this parameter are:</p>

 * </ul>

 *   <li>TL ({@link ATT.Location Terminal Location})</li>
 * </ul>
 * @param {String} [options.redirectUrl] Specifies the URL where the subscriber's browser is redirected following completion of the authorization process.  * <strong>Examples</strong>
 *
 * The following example gets the end user authorization for the Location APIs for the APP key <strong>1234567890</strong>, using the default redirect URL.
 *
 *     obtainEndUserAuthorization({
 *        'contentType' : 'application/json',
 *        'clientId' : '1234567890',
 *        'scope' : 'TL'
 *     }, function(data) {
 *           success Callback
 *     }, function(error) {
 *           error Callback
 *     });
 *

/**
 * @method authorize
 * Specifies the security credentials for the session.
 * 
 * This method must be called directly using the ATT namespace, as shown in the following syntax block.
 *
 * <b>Syntax</b>
 * 
 * <code>Att.authorize('<em>apiKey</em>','<em>secretKey</em>','<em>scope</em>','<em>grantType</em>','<em>oAuthCode</em>')</code>
 *
 * @param {String} accessKey Specifies the API key that is assigned to the application when the developer creates an application account at <a href="https://developer.api.att.com">https://developer.api.att.com</a>.
 * @param {String} secretKey Specifies the secret key that is assigned to the application when the developer creates an application account at <a href="https://developer.api.att.com">https://developer.api.att.com</a>.
 * @param {String} scope Specifies the services for which the application is requesting the end-user's consent to use. The following values are defined for this parameter. Multiple values must be separated by a comma (,).
 * <ul>
 *   <li>IMMN and MIM ({@link ATT.IMMN In-app Messaging from Mobile Number})</li>
 *   <li>TL ({@link ATT.Location Terminal Location})</li>
 * </ul>
 * @param {String} grantType Specifies the mechanism used to obtain the access token.
 * The following values are defined for this parameter.
 * <ul>
 *   <li>authorization_code
 *     <br/>
 *     Use when Web-server client applications use the authorization code to obtain the access token.</li>
 *   <li>client_credentials
 *     <br/>
 *     Use when autonomous client applications use the API key and its secret to obtain the access token.</li>
 *   <li>refresh_token
 *     <br/>
 *     Use when client applications of both previous types use need to obtain a new access token after the initial access token expires.</li>
 * </ul>
 *

 *
 * The following example gets the end user authorization for the Location APIs for the APP key <strong>1234567890</strong>, using the default redirect URL.

 *
 */
